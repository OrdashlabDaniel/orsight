## 目标

做一个专门处理站点 POD 抽查截图的网页端 AI Agent，供站点同事每天上传截图、自动提取、人工复核、导出 Excel。

核心目标：

- 零乱填
- 可追溯
- 易上手
- 支持多人每天重复使用

## 产品流程

1. 用户进入网页
2. 选择日期
3. 上传当天截图文件夹或多张图片
4. AI 按固定规则逐图提取字段
5. 规则引擎自动校验
6. 对高风险截图打上 `待复核`
7. 用户在网页上逐条确认或修正
8. 导出 Excel
9. 保存当天审计记录

## 网页端必须具备的功能

### 1. 上传区

- 支持拖拽上传多张图片
- 自动显示文件名、上传时间、图片缩略图

### 2. 识别结果表格

每条结果显示：

- 日期
- 抽查路线
- 抽查司机
- 运单数量
- 未收数量
- 错扫数量
- 识别置信度
- 是否需要人工复核
- 原始截图预览

### 3. 复核工作台

点击一条记录后，右侧显示原图，并高亮：

- 司机姓名区域
- 路线区域
- 应领件数区域
- 未领取区域
- 错分数量区域

用户可以直接改值并保存。

### 4. 风险拦截

以下情况必须阻止直接导出：

- 路线为空
- 司机为空
- 运单数量为空
- 使用了站点车队代替路线
- 多任务截图但有任务未完整识别
- 任一字段来自推断而非截图
- 有 `待复核` 记录未处理

### 5. 导出

- 导出标准 Excel
- 保持与站点现有表头一致
- 可额外导出复核报告 JSON 或 CSV

## 推荐技术方案

### 前端

- `Next.js`
- `TypeScript`
- `Tailwind CSS`
- 表格组件可用 `AG Grid` 或 `TanStack Table`

### 后端

- `FastAPI` 或 `Next.js API routes`
- 图片上传到对象存储或本地文件系统
- 任务队列可选 `Celery` / `RQ` / `BullMQ`

### AI 识别层

- 多模态大模型识别单张截图
- 强制使用固定提示词和 JSON 输出
- 每张图片保留原始识别结果，方便复盘

### 规则引擎

把这次整理出来的业务规则写成独立模块：

- 字段映射校验
- 空值校验
- 多任务校验
- 推断值拦截
- 重复截图检测

### 导出层

- 使用 `openpyxl` 生成 Excel
- 支持导出站点标准模板

## 零错误率落地方法

零错误率不能只靠模型准确率，必须靠“模型 + 规则 + 人工复核”的闭环。

建议采用三层机制：

1. 第一层：AI 提取
   - 只负责读图和填结构化 JSON

2. 第二层：规则引擎
   - 不允许站点车队进路线列
   - 不允许 `已领` 进入 `运单数量`
   - 不允许推断第二条任务数量

3. 第三层：人工复核
   - 所有多任务截图
   - 所有模糊截图
   - 所有低置信度截图
   - 所有被规则引擎拦截的截图

## 推荐开发阶段

### 第一阶段：内部工具 MVP

- 上传图片
- AI 提取
- 校验规则
- 网页端手工改值
- 导出 Excel

目标：先让您自己和 1 到 2 位同事能每天用。

### 第二阶段：站点共享版

- 用户登录
- 历史记录
- 复核状态流转
- 当日任务列表

目标：让整个站点都能使用。

### 第三阶段：正式 Agent

- 一键上传整天截图
- 自动分类与批量识别
- 自动比对历史异常
- 自动生成日报

## 现在最适合的下一步

先做一个本地可运行 MVP：

- 用固定提示词识别图片
- 用 `pod_fill_guard.py` 校验
- 生成 Excel
- 同时输出复核报告

等这套流程稳定后，再把同样逻辑搬到网页端。
