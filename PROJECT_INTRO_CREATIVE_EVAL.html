Creative Evaluation Brief

POD Screenshot Audit Tool

一个面向站点日常运营场景的 AI 审核与填表工具。它把“截图识别、风险拦截、人工复核、Excel 导出、训练样本积累”整合为一个可落地的闭环流程。

项目定位

该项目不是通用 OCR 工具，而是一个高度垂直、围绕 POD 抽查业务规则设计的智能工作台。核心目标不是“尽量识别更多”，而是“在保证准确率与可追溯性的前提下提升录入效率”。

AI + 规则引擎 高风险先拦截 人工复核闭环 本地可运行 MVP

评估摘要

产品类型 运营提效型垂直 AI 工具
目标用户 站点审核人员 / 运营同事
当前阶段 内部单用户 MVP，已可运行
核心价值 减少人工重复录入与误填风险

1. 产品介绍

项目聚焦 POD 截图抽查这一高频但高度依赖人工经验的工作场景，把原先分散的“看图、抄表、复核、导出”步骤整合到统一页面中。

解决的问题

  • 人工从截图录入表格效率低，容易疲劳出错。
  • 截图中字段位置相近，容易误把站点车队、实领件数、已领等字段写进错误列。
  • 多任务截图、模糊截图、遮挡截图需要人工兜底，普通 OCR 难以可靠处理。
  • 传统工具缺少“可复核、可追踪、可沉淀样本”的产品闭环。

项目给出的答案

  • 批量上传截图，由 AI 自动提取结构化记录。
  • 通过业务规则校验与多次一致性检查，主动拦截高风险结果。
  • 让用户在同一界面里查看原图、编辑表格、重新识别、打开标注。
  • 把人工修正过的结果沉淀成训练样本，持续提高后续识别稳定性。

2. 产品流程

输入

用户批量上传 POD 截图，支持拖拽和逐张预览。

识别

AI 按固定提示词提取日期、路线、司机、运单数量、未收数量、错扫数量等核心字段。

拦截

系统对结果进行多轮一致性校验、字段来源校验、计数区域二次验证，主动标记待复核记录。

复核

用户可在线修改表格内容，并打开独立图片查看窗口或标注工作台进行人工确认。

沉淀

复核后的图片和框选结果可保存到训练池，用作后续 few-shot 示例。

输出

最终导出 Excel，满足站点日常抽查结果提交需求。

3. 产品亮点

不是“识别优先”，而是“可信优先”

该工具最有价值的点在于，它把“错误拦截”放在“结果输出”之前。比如当 `运单数量` 不能确认来自 `应领件数` 时，系统不会保留一个可能错误的数字，而是主动清空并要求人工复核。

将人工动作设计进产品闭环

项目没有试图完全替代人工，而是把“人工查看原图、改值、标注、再次识别”设计成系统流程的一部分，因此更适合高准确率要求的业务环境。

4. 技术实现

当前技术方案以快速落地和稳定迭代为导向，采用 Next.js 单体式 Web 应用结构，前后端协同成本低，适合 MVP 阶段快速推进。

层级 当前方案 说明
前端框架 Next.js 16 + React 19 + TypeScript 用于构建上传区、在线表格、图片查看弹窗、标注工作台等交互界面。
样式系统 Tailwind CSS 4 用于快速构建信息密度较高但仍保持清晰层次的后台工具界面。
AI 接口 OpenAI 风格 `chat/completions` 支持视觉理解模型识别截图内容，并强制返回 JSON。
模型策略 `gpt-5-mini` + `gpt-5` 批量场景优先成本效率，复核场景优先更强模型能力。
图像处理 `sharp` 用于裁剪计数区域，执行二次核验。
表格导出 `xlsx` 生成最终 Excel 文件，直接衔接现有业务交付形态。
训练样本池 本地 JSON + 本地图片目录 当前是本地文件式训练池，用于 few-shot 示例增强，不是真正在线微调。

核心技术策略

  • 单张图片执行 4 次识别，结果不一致时直接标记风险。
  • 要求模型显式返回 `totalSourceLabel`，验证运单数量是否来自 `应领件数`。
  • 额外裁剪 `应领件数`、`实领件数`、`已领` 区域进行二次数字核验。
  • 将人工校正过的案例回流到训练池，形成持续提准机制。

为什么这个技术方案有价值

  • 不是单纯依赖大模型“看图猜答案”，而是把业务规则硬编码进验证链路。
  • 技术架构足够轻，适合快速迭代业务规则与提示词。
  • 未来可平滑迁移到云存储、数据库、用户体系和订阅模型。

5. UX / 交互设计思路

面向业务人员的低学习成本设计

  • 左侧上传、右侧表格的双栏结构，符合“先看图、再改表”的工作心智。
  • 识别后结果直接落到可编辑表格，无需切换到额外编辑页面。
  • 高风险记录高亮显示，让用户优先处理问题项而不是逐条全量检查。

独立的图片查看与标注体验

  • `查看图片` 是独立浮层，靠近上传区，不遮挡主表格，方便边看边修。
  • `打开标注` 是另一套独立工作台，用于训练样本标框，避免和普通查看混淆。
  • 图片查看支持缩放和平移，适合检查细节数字与模糊区域。

UX 评价

从创意评估角度看，这个项目的 UX 亮点不在炫技，而在于对真实工作流的理解非常具体。它把“错误最容易发生在哪一步、用户最需要在哪一步看到原图、什么时候该让系统拦截而不是放行”都映射到了交互上，因此具备较强的场景贴合度。

7. 当前风险与发展空间

当前风险

  • 训练池和图片资源目前仍是本地文件存储，尚未云端化。
  • 缺少用户体系、历史记录、权限与计费能力。
  • 业务规则强依赖当前 POD 截图结构，泛化到其他站点时需要再适配。

发展空间

  • 演进为站点级审核平台，支持多人协作与记录沉淀。
  • 接入数据库后可形成历史异常对比、每日审核台账与质检报告。
  • 可进一步抽象为“规则驱动的业务型视觉审核产品框架”。

8. 创意评估结论

这是一个场景切得很准的垂直 AI 产品创意。它的价值不在于做出一个“更通用的识图工具”，而在于围绕具体业务错误模式，把 AI、规则、交互、人工复核和样本沉淀组合成一个高可信工作流。

从创意成熟度看，项目已经不只是概念验证，而是进入了可展示、可试用、可继续产品化的阶段。对于内部评估、立项讨论、产品包装或后续融资材料准备，它具备较好的基础。

如果后续补齐云存储、登录体系、数据库和使用追踪，这个项目有机会从内部效率工具升级为面向站点运营场景的轻量 SaaS 产品。
